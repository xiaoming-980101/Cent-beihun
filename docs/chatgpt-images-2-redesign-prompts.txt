# Cent / YueWed 页面重设计生图提示词与复刻工作流

本文档用于把当前项目所有可访问页面重新生成一套视觉设计稿，再交给 Claude Code 按项目现有技术栈复刻。

适用模型：`chatgpt-images-2`

项目定位：婚礼筹备记账 PWA。核心体验是把记账、礼金、宾客、预算、任务、统计、AI 助手和资料管理整合成一个温柔但高效的婚礼项目工作台。

技术落地约束：React 19、Vite、TypeScript、TailwindCSS 4、CSS Variables、Radix UI、shadcn/ui、lucide-react、ECharts、Motion、Zustand。请保留现有信息架构和业务字段，不要把它做成纯营销页。

## 与当前项目的关系

这份提示词不是要求把当前项目改成一个全新的婚礼 App，而是在现有 Cent 记账 PWA 上做视觉重设计。当前项目有两层产品身份：

1. 原 Cent 固有能力：多人协作记账、账本、账单新增/编辑/详情、搜索筛选、统计分析、分类标签、多币种、周期记账、预算、数据导入导出、GitHub/Gitee/WebDAV/S3/离线同步、AI 助手、语音/键盘快速记账、主题和语言设置。
2. 当前 YueWed 婚礼扩展能力：婚礼首页、任务、任务日历、礼金簿、亲友管理、婚礼预算、智能筹备建议、资料整理中心。

生图和复刻时必须保留这两层身份。不要把原 Cent 的记账、同步、设置、AI 和数据管理能力画没。

下面这些功能虽然不一定都是独立路由，但它们是当前项目真实存在的核心功能，必须在页面提示词、弹窗提示词或设置页里体现。

每一页生图前，先粘贴这一段，再追加对应页面提示词。

```text

视觉方向：现代、细腻、轻奢、清晰、高级婚礼工具感。不要廉价粉色，不要大面积渐变噪音，不要卡通插画，不要过度装饰。整体要有产品级信息密度，像 Apple Health、Linear、Craft、Structured 和高端婚礼策划手册的结合。

- 主色：#F472B6, #EC4899

- 边框：#E5E7EB, #F3E8FF

布局要求：

- 内容必须真实像可用产品界面，包含中文 UI 文案和合理数字。
- 卡片圆角控制在 16-28px，但内部小控件不要全部做成大圆胶囊。
- 标题、数字、标签、按钮、图表、列表层级要清晰。
- 不要在页面里写说明性文字解释设计功能，不要出现“这是一个...”这类设计说明。

```

## 日夜双版本要求

每个生产页面都必须生成白天版和黑夜版。白天版用于默认浅色主题，黑夜版用于深色主题落地。

### 白天基础版色调

白天版不是纯白极简，也不是粉色主题铺满全屏。它应该是“暖白纸张 + 雾粉品牌 + 冰蓝功能 + 少量紫色仪式感”的基础调。

```text
白天基础版色调：
- 页面背景：#FAFAFA / #FFF7FB / #F8FAFC，优先暖白和极浅粉白。
- 主卡片：#FFFFFF，允许极轻粉紫渐变 #FFF7FB -> #F8F3FF。
- 主要按钮：#F472B6 -> #A855F7 的克制渐变，或纯 #EC4899。
- 辅助按钮与信息区：#EFF6FF / #DBEAFE / #F3E8FF。
- 成功：#22C55E；警告：#F97316；危险：#EF4444。
- 主文字：#111827；正文：#374151；辅助：#6B7280；弱提示：#9CA3AF。
- 边框：#E5E7EB / #F3E8FF，阴影要轻，不要厚重投影。
- 金色只能作为极细分割线或小图标点缀，不能成为主色。
```

白天版禁止：

```text
不要大面积纯粉背景；不要廉价婚庆红；不要卡通爱心满屏；不要高饱和紫蓝渐变铺底；不要过多发光效果。
```

### 黑夜基础版色调

黑夜版不是纯黑，也不是霓虹赛博。它应该是“深蓝灰底 + 深梅紫氛围 + 低亮度粉紫品牌光 + 清晰文字对比”的基础调。

```text
黑夜基础版色调：
- 页面背景：#0F172A / #111827 / #171122，优先深蓝灰和深梅紫混合。
- 主卡片：#1F2937 / #231A2E / rgba(255,255,255,0.06)。
- 次级卡片：#111827 / #1A1323 / rgba(255,255,255,0.04)。
- 主要按钮：#F472B6 -> #A855F7，但亮度降低，避免荧光。
- 辅助按钮与信息区：rgba(59,130,246,0.16)、rgba(168,85,247,0.16)、rgba(244,114,182,0.14)。
- 成功：#4ADE80；警告：#FDBA74；危险：#F87171。
- 主文字：#F9FAFB；正文：#E5E7EB；辅助：#CBD5E1；弱提示：#94A3B8。
- 边框：rgba(255,255,255,0.10) / rgba(244,114,182,0.16)。
- 阴影用深色层级和细边框表达，不要依赖大面积外发光。
```

黑夜版禁止：

```text
不要纯黑背景；不要霓虹赛博风；不要蓝紫荧光过曝；不要把图表坐标和标签压暗到看不清；不要把所有卡片都做成半透明玻璃。
```

追加到每个页面提示词末尾：

```text
请为这个页面生成 2 张独立截图：
1. 白天版：浅色主题，暖白背景，雾粉/浅紫/冰蓝点缀，整体明亮、清爽、适合日常使用。
2. 黑夜版：深色主题，不是纯黑，使用 #111827、#1F2937、深梅紫、深蓝灰作为底色，保留粉紫品牌色和功能色。文字对比度必须清楚，卡片边框和阴影在暗色中仍可辨识。

两个版本布局、信息层级、组件位置必须一致，只调整色彩、材质、阴影和图表背景。不要让黑夜版变成霓虹赛博风，也不要让白天版过曝。
```

## 弹窗与覆盖层生图总要求

除了页面本体，每个涉及新增、编辑、筛选、确认、设置、详情的页面都要额外生成关键弹窗/抽屉状态。弹窗设计必须能指导 `ResponsiveDialog`、`ResponsiveConfirmDialog`、表单弹窗、底部 Sheet 和桌面居中弹窗的统一落地。

弹窗通用提示词：

```text
为同一页面设计关键弹窗状态截图，移动端 390x844。背景保留当前页面并加半透明磨砂遮罩，弹窗使用底部 Sheet 或全屏表单，根据内容复杂度选择。

弹窗要求：
- 保留页面背景上下文，背景轻微压暗或模糊，但仍能看出来自哪个页面。
- 移动端复杂表单使用全屏 Sheet，简单确认或筛选使用半屏底部 Sheet。
- 弹窗顶部包含拖拽手柄、标题、说明、关闭按钮或返回按钮。
- 表单字段分组清楚，输入框、日期选择、金额、下拉、单选、多选、备注都要有真实内容。
- 底部固定操作区包含取消和确认按钮，确认按钮使用品牌渐变或页面主色。
- 展示 loading/disabled/error 必要状态时，按钮和字段要有明确反馈。
- 白天版和黑夜版都要生成。
- 不要让底部按钮遮挡表单内容，注意安全区。
```

## 章节结构

第 1 章：核心产品闭环，共 10 个生产页面。每页包含白天版、黑夜版；有弹窗/抽屉的页面额外生成弹窗状态。

1. 首页 `/`
2. 任务列表 `/tasks`
3. 任务日历 `/tasks/calendar`
4. 工具箱 `/tools`
5. 礼金簿 `/tools/gift-book`
6. 亲友管理 `/tools/guests`
7. 婚礼预算 `/tools/wedding-budget`
8. 智能筹备建议 `/tools/planner-assistant`
9. 资料整理中心 `/tools/resource-center`
10. 统计分析 `/stat` 与 `/stat/:id`

第 2 章：账本通用能力与设置，共 2 个生产页面，另含 1 个内部调试页。生产页面同样包含白天版、黑夜版和关键弹窗状态。

11. 搜索记录 `/search`
12. 设置 `/settings`
13. 弹窗系统测试 `/modal-test`，仅用于内部验证，不建议作为正式视觉批次生成。

## 第 1 章：核心产品闭环

### 1. 首页 `/`


## 批量生成建议

每 10 页作为一章生成时，建议按以下顺序：

1. 先生成第 1 章第 1 页首页，确定整体视觉语言。
2. 用首页成图作为后续 9 页的视觉参考，让模型保持一致。
3. 第 1 章剩余页面逐页生成，不要一次让模型出 10 张，否则细节会变泛。
4. 第 2 章用第 1 章已确定的组件语言延展，搜索和设置要更工具化。
5. 每个页面先生成白天/黑夜主屏，再生成该页关键弹窗与状态。
6. 对每页至少生成 2 个设计变体：`A 信息密度优先`、`B 情绪氛围优先`。最终复刻时选择更可落地的一版。

追加到每个页面提示词末尾的版本控制语句：

```text
生成四个主屏版本：
1. A 版白天：信息密度更高，更像真实生产应用。
2. A 版黑夜：沿用 A 版布局，转换为深色主题。
3. B 版白天：更有婚礼情绪和高级仪式感，但仍然必须可落地。
4. B 版黑夜：沿用 B 版布局，转换为深色主题。

选定主屏方向后，再按“每页必须补充生成的弹窗、状态与细节”清单生成对应覆盖层。所有覆盖层都要白天版和黑夜版。
```

## 复刻验收清单

视觉验收：
- 12 个生产页面风格统一，首页、工具箱、任务、预算、统计有各自重点。
- 移动端 390x844 不横向溢出，按钮文字不挤压。
- 桌面端 1440x1024 不只是简单拉伸，内容宽度有上限。
- 状态色一致：完成绿、进行中蓝、待办灰、警告橙、危险红。
- 金额、日期、状态、操作按钮有稳定位置。

交互验收：
- 所有新增、编辑、删除、筛选、排序、切换、弹窗仍可用。
- `/stat/:id` 保存筛选视图仍能打开。
- 搜索页多选、批量编辑、批量删除仍可用。
- 移动端弹窗不会遮挡底部操作按钮。

工程验收：
- `pnpm run lint` 通过。
- 相关测试通过，至少覆盖 tasks、tools、settings/search 已有测试。
- 不引入新的 UI 库，优先使用 lucide-react、Radix、现有 shadcn/ui。
- 不把设计图里的文字硬编码成假数据，仍使用当前 store 数据。
